For now - Built-in Functions

There is no standard library yet. Instead, there is currently a suite of built-in relish methods which provide useful operations on relish primitive data types, such as String, Numeric, Bool, Time, List, Channel. These functions are implemented as "native" functions, written in Go. Many are thin wrappings of functions from several of Go's standard library packages, such as the "strings" package, the "time" package etc. Others are wrappings of Go primitive operators such as arithmetic and logical operators, and slice expressions.

The currently available Built-in Functions are listed in the relish Syntax and Semantics By Example reference manual.

Roadmap

The plan is to lift a subset of Go standard library packages into relish by providing thin adapter wrappers for the Go functions. Wrapper functions generally just cast the Go primitive data type arguments into equivalent data types which implement the RObject interface, which every relish data object implements.

A small number of methods will remain as relish language standard built-in methods, available for use in any relish package without importing a package of methods. More of the "lifted" standard library methods, however, will be placed in separate relish packages. The library packages will then have to be individually imported into any new package in order to use the standard library package method.
